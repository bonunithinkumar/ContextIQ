## 🌟 What Is This Project?

This is a **Retrieval-Augmented Generation (RAG)** system built in Python. In simple terms:

1. **Feed it your documents** — PDFs, text files, Word docs, CSVs, Excel files, or JSON
2. **Ask it a question** — e.g., `"What is Naive Bayes?"`
3. **Get a smart answer** — Generated by an AI model, grounded in your documents

Unlike asking ChatGPT directly (which answers from memory), this system:
- Searches YOUR files for relevant content first
- Uses that content as context for the AI
- Produces answers that are directly tied to your documents

**Why RAG?** Because general AI models don't know what's in your private files. RAG bridges that gap.

---

## ⚙️ How It Works

```
📄 Your Documents (PDFs, TXTs, etc.)
        ↓  [Data Loading]
📋 LangChain Document Objects
        ↓  [Chunking — 1000 chars, 200 overlap]
✂️  Small Text Chunks
        ↓  [Embedding — all-MiniLM-L6-v2]
🔢 Vectors (384 numbers per chunk)
        ↓  [Indexing]
💾 FAISS Vector Store (saved to disk)

         ↑ Build Once, Reuse Always

❓ User Question
        ↓  [Embed Question]
🔢 Query Vector
        ↓  [FAISS Similarity Search — top 3]
📌 Most Relevant Chunks
        ↓  [Build Prompt + Send to Groq]
✅ AI-Generated Answer (from llama-3.1-8b-instant)
```

**Two Phases:**
- **Indexing Phase** (runs ONCE, saves to `faiss_store/`) — Processes all documents and builds the searchable database
- **Query Phase** (runs EVERY TIME) — Fast similarity search + LLM answer generation

## 🚀 Setup & Installation

### Prerequisites

- Python 3.9 or higher
- A free [Groq API Key](https://console.groq.com/) (takes 30 seconds to get)
- ~500 MB disk space (for the embedding model)

### Step 1: Clone or download the project

```bash
cd ~/Desktop/RAG_Practice
```

### Step 2: Create a virtual environment

```bash
python -m venv .venv
source .venv/bin/activate        # Mac/Linux
# .venv\Scripts\activate         # Windows
```

### Step 3: Install dependencies

```bash
pip install -r requirements.txt
```

> The first install will also download the `all-MiniLM-L6-v2` model (~80 MB) automatically when you first run the app.

### Step 4: Add your Groq API key

Edit the `.env` file:

```
GROQ_API_KEY = "your_groq_api_key_here"
```

Get your key at: [https://console.groq.com](https://console.groq.com)

## 🔍 Source Citations (Reliability Proof)

To verify the accuracy of the AI-generated answers, the system displays references/citations at the bottom of the output showing exactly where the source context was retrieved:

* **PDF Files**: Displays the document name and `1-indexed` page number (e.g., `Doc: Machine_Learning_Basics.pdf, Page: 14`).
* **Text Files**: Displays the file name and the computed start line number of the matching chunk (e.g., `Doc: machine_learning.txt, Line: 12`).
* **CSV Files**: Displays the file name and row number (e.g., `Doc: dataset.csv, Row: 5`).
* **JSON Files**: Displays the file name and sequence index (e.g., `Doc: data.json, Sequence: 2`).

## ⚙️ Configuration

All key parameters are in your source files:

## 🏛️ Architecture Deep Dive

### Module Responsibilities

#### `src/data_loader.py` — The Librarian
Scans all subdirectories of `data/` and loads every supported file into LangChain `Document` objects. Each `Document` has:
- `page_content` — the raw text
- `metadata` — source file, page number, etc.

#### `src/embedding.py` — The Translator
Two responsibilities:
1. **Chunking** — Uses `RecursiveCharacterTextSplitter` to split long documents into small, focused chunks (1000 chars, 200 overlap)
2. **Embedding** — Uses `SentenceTransformer("all-MiniLM-L6-v2")` to convert each chunk into a 384-dimensional float vector

#### `src/vectorstore.py` — The Database
Wraps Facebook's FAISS library:
- **Build**: Takes embeddings → creates `IndexFlatL2` → saves to disk
- **Load**: Reads `faiss.index` + `metadata.pkl` from disk
- **Query**: Embeds a text query → runs FAISS nearest-neighbor search → returns top-k matching chunks

#### `src/search.py` — The Orchestrator (RAG Core)
Combines vector search + LLM generation:
1. Calls `vectorstore.query(question, top_k=3)` → gets relevant text chunks
2. Builds a prompt: `"Given this context: [chunks], answer: [question]"`
3. Sends to Groq API (`llama-3.1-8b-instant`)
4. Returns the AI-generated answer

### Data Flow Diagram

```
app.py
  │
  ├─→ data_loader.py::load_all_documents()
  │         │ returns List[Document]
  │
  ├─→ vectorstore.py::FaissVectorStore
  │         │
  │         ├─→ embedding.py::EmbeddingPipeline
  │         │         ├── chunk_documents() → List[Document chunks]
  │         │         └── embed_chunks() → np.ndarray (N×384)
  │         │
  │         ├── add_embeddings() → faiss.IndexFlatL2
  │         ├── save() → faiss_store/faiss.index + metadata.pkl
  │         └── load() → restore from disk
  │
  └─→ search.py::RAGSearch
            │
            ├── vectorstore.query(question) → top-k chunks
            ├── build prompt with chunks
            └── llm.invoke(prompt) → final answer
```

## 🛠️ Troubleshooting

### `ModuleNotFoundError: No module named 'src'`
Run from the project root directory:
```bash
cd ~/Desktop/RAG_Practice
python app.py   # Not python src/app.py
```

### `GROQ_API_KEY not found`
Make sure your `.env` file exists and has the key:
```
GROQ_API_KEY = "gsk_your_key_here"
```

### FAISS index is stale / old documents being returned
Delete and rebuild:
```bash
rm faiss_store/faiss.index faiss_store/metadata.pkl
python app.py
```

### PDF not loading
Make sure `pypdf` is installed:
```bash
pip install pypdf pymupdf
```

### Slow first run
Normal! The first run downloads the embedding model (~80 MB) and processes all your PDFs. Subsequent runs load from cache and are much faster.

## 📖 Key Concepts Reference

| Concept | Explanation |
|---------|-------------|
| **Embedding** | Text converted to numbers; similar meanings = close numbers |
| **Vector** | A list of numbers representing a piece of text |
| **Chunking** | Breaking long documents into smaller 1000-char pieces |
| **FAISS** | Facebook's library for fast vector similarity search |
| **top_k** | "Return the K most similar results" (default: 3) |
| **LLM** | Large Language Model — the AI that writes the final answer |
| **Groq** | Cloud service that runs open-source LLMs at very high speed |
| **RAG** | Retrieval + context + generation: grounding AI in your documents |
